# PRD 合并 Prompt

将以下材料整合为一份完整的产品需求文档（PRD）：

## 输入材料

1. 结构化会议纪要（决策和方向）
2. 行动项清单（任务分解）
3. 技术方案对比（技术选型依据）

## PRD 结构要求

1. **背景与目标**：为什么做这个版本
2. **核心功能**：按优先级列出功能需求
3. **功能详情**：每个功能的验收标准
4. **技术方案**：关键技术选型及理由
5. **里程碑**：时间线和交付节点
6. **风险与缓解**：已知风险和应对策略

## 写作规则

- 验收标准用 Given-When-Then 格式
- 时间线与行动项清单保持一致
- 技术选型引用对比分析的结论
