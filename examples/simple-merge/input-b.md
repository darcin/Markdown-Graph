# 技术调研 - 搜索方案对比

## 背景
当前搜索基于 SQL LIKE 查询，性能和精度都不理想。需要评估替代方案。

## 方案对比

| 方案 | 优点 | 缺点 | 成本 |
|------|------|------|------|
| Elasticsearch | 功能强大，生态成熟 | 运维复杂，资源占用高 | 高 |
| MeiliSearch | 轻量，易部署，中文友好 | 功能不如 ES 全面 | 低 |
| PostgreSQL FTS | 无需额外组件 | 中文分词支持弱 | 无 |

## 初步结论
建议采用 MeiliSearch，理由：
1. 团队规模小，运维能力有限
2. 数据量在百万级以下
3. 中文搜索开箱即用
